# Release Checklist

Status values: `complete`, `blocked`, or `deferred`.

| Area | Status | Evidence |
| --- | --- | --- |
| Contract package | complete | `packages/contracts/package.json`, `packages/contracts/CHANGELOG.md`, generated schemas/examples, `docs/consumers.md`. |
| TypeScript build | complete | `npm run build`. |
| Contract/unit tests | complete | `npm test`. |
| Database migrations | complete | `npm run test:db`, `npm run migrate:dev`, `infra/migrations/`. |
| Queue/status integration | complete | `npm run integration:localstack`, `tests/integration/localstack/harness.mjs`. |
| Return/retry integration | complete | `npm run test:integration`, `tests/returns/`, `tests/integration/retry/`. |
| Deployment templates | complete | `npm run templates:check`, `infra/cdk/cdk.out/`. |
| Sandbox evidence bundle | complete | `docs/release/1.0.0/`, generated by `node scripts/release-evidence.mjs --version 1.0.0`. |
| Restricted-production evidence | deferred | Owner: release owner. Review date: 2026-06-05. Requires explicit authorization and redaction rules. |
| Git tag/publication | deferred | Owner: release owner. Do not tag or publish until explicitly requested. |

## Security

- Certificate custody is metadata-only in the database. Certificate bytes,
  private keys, real certificates, and production payloads are forbidden in git.
- Signing tests use generated local keys only.
- XML hardening rejects DTD, entity declarations, stylesheets, and external
  identifiers before signing or SOAP submission.
- Generated templates include KMS encryption, Secrets Manager placeholders,
  service roles, queue encryption, and private database security groups.
- Non-production SOAP/template paths cannot target `gov.br` endpoints.

## Data Protection

- The service database uses schema `esocial` and tenant RLS with
  `app.current_tenant_id`.
- Worker cross-tenant access is limited to the `esocial_worker` role.
- SGP references are opaque identifiers in envelopes, not foreign keys.
- Audit and status history are append-oriented; tests reject direct mutation of
  history tables.

## Observability

- Logs use stable fields documented in `docs/operations.md`: request id,
  correlation id, tenant id, event class, batch id, protocol, receipt, and
  idempotency key.
- Metrics use stable names: `esocial.accepted`, `esocial.rejected`,
  `esocial.retry`, `esocial.dlq`, `esocial.timeout`,
  `esocial.soap_latency_ms`, `esocial.queue_age_ms`, and
  `esocial.parser_failures`.
- Templates include CloudWatch alarms and dashboard resources for the metric
  names above.

## Migrations

- Migrations are forward-only SQL under `infra/migrations/`.
- `npm run migrate:dev` applies the full chain from zero to a local or explicit
  database URL.
- The deployment template includes a CodeBuild migration hook using the same
  migration command surface.

## Rollback

- Pause SGP request publishing before deployment rollback.
- Drain or pause FIFO request/retry queues; do not delete DLQ evidence.
- Roll back compute/template deployment first, then reconcile pending database
  statuses from audit and spool evidence.
- Do not replay accepted official events through SGP historical paths.

## Evidence Retention

- Deterministic sandbox evidence is retained under `docs/release/<version>/`.
- Operational audit evidence is retained in `esocial.audit_event_log` and the
  configured audit/EventBridge sink.
- Restricted-production evidence must be redacted before commit and retained
  according to the release owner policy.
