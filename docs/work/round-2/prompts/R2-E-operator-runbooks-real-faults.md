# R2-E - Operator Runbooks for Real Fault Modes

> This batch updates runbooks from real Round 2 observations. It does not call
> official eSocial endpoints.

## Preconditions

- R2-B and, if applicable, R2-C evidence is available.
- SRE owner and release owner have reviewed observed faults.
- All evidence used by this batch is redacted.

## Primary Write Scope

- `docs/operations.md`
- `docs/release-checklist.md`
- `docs/sgp-migration.md`
- `docs/release/0.3.0/runbooks/**`

## Do Not Touch

- Runtime code except for documentation links generated by existing scripts.
- Unredacted logs, payloads, certificates, or production data.

## Tasks

1. Map each observed real fault mode to:
   - detection signal
   - first triage step
   - owner escalation
   - customer/SGP-facing status
   - retry, replay, DLQ, or rollback action
   - audit evidence to retain
2. Cover at minimum:
   - portal account/auth failure
   - certificate expiry or wrong certificate
   - TLS pin mismatch
   - SOAP timeout
   - official throttling
   - regulatory rejection
   - malformed official return
   - unknown official code
   - DLQ replay
   - restricted-production rollback
3. Link runbooks to metrics, alarms, log fields, and trace identifiers.
4. Update SGP migration notes if any official status behavior changed.
5. Add an operator checklist for ending a real-service connectivity window.

## Verification

Run:

```bash
npm test
npm run lint
```

If a docs consistency script exists, run it and record the command.

## Exit Criteria

- Runbooks reflect implemented behavior and real observed faults.
- Every fault has an owner escalation and customer/SGP-facing status.
- No runbook tells an operator to inspect raw certificate material or unredacted
  personal data.

## Report

Report updated runbooks, covered fault modes, owner escalations, and any
operator gaps that block go-live.
